# 学习记录

这里记录我日常学到的东西。内容不一定完整，结论也可能随着实践继续修改。

## Terminal

- 终端里的命令运行在进程中。排查问题时，要关注父子进程、前台进程组和信号传播。
- Login shell 与 interactive shell 读取的配置文件不同，环境变量问题通常需要先确认启动方式。

## CLI

- `stdout`、`stderr` 和退出码共同组成 CLI 的接口，输出不仅给人看，也会被脚本和 Agent 消费。
- 破坏性命令应该提供 `--dry-run`，让用户在执行前确认目标和影响范围。

## Desktop

- 桌面应用需要划清主进程、渲染进程和系统能力之间的边界。
- 深链接属于外部输入，需要处理参数校验、冷启动和已有窗口等情况。

## OpenClaw

- 学习 OpenClaw 时，实验结论应该同时记录版本、运行环境和复现步骤。
- 先梳理输入、状态、工具、权限和输出，再研究具体配置。

## Tool

- 工具的名称、描述、参数和错误信息会直接影响模型如何使用它。
- 工具返回值应该包含下一步决策真正需要的信息，减少模型猜测和重复调用。

## Sandbox

- Sandbox 管理的不只是文件目录，还包括网络、进程、设备、凭证等能力。
- 可复现的隔离环境需要记录依赖版本、环境变量、工作目录和副作用。

## Coding agents

- 清晰的任务边界、验证方式和完成条件，比堆叠很长的提示词更重要。
- 构建、测试、截图和差异检查应该进入 Agent 的反馈循环，而不是最后才补。

## Digital agents

- Digital agent 的连续性来自可管理的身份、目标、记忆和状态，而不是无限增长的上下文。
- 当权限不足或事实不确定时，可靠的交接通常比勉强追求完全自治更重要。
